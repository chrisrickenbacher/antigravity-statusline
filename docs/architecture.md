# Architecture

This document illustrates the design, data pipeline, and JSON schema of the Antigravity Status Line rendering engine and background daemon.

## Data Flow & Components

```mermaid
graph TD
    subgraph Hot Path (Real-time CLI Interaction)
        AG["Antigravity CLI"] -->|Stdin State Payload| SL["statusline CLI Renderer"]
        SL -->|Renders ANSI Formatted Output| AG
        SL -->|Append turn stats| SL_Logs[("Local Session Logs\nusage_&lt;conv_id&gt;_&lt;date&gt;.jsonl")]
        SL_Logs -->|Read & Sum session cached tokens| SL
    end

    subgraph Background Path (Asynchronous Daemon, 1 min poll)
        Daemon["agy-statusline-daemon Background Service"] -->|Aggregate Today's Logs| SL_Logs
        Daemon -->|Download Pricing Once/24h| Pricing_Repo["Remote GitHub Repository"]
        Daemon -->|Write Today's Stats| API_Usage[("~/.gemini/.../api_usage.json")]
        API_Usage -->|Read Today's Total Cost & Health| SL
        Pricing_Repo -->|Cache Rates| Pricing_Cache[("pricing_cache.json")]
        Pricing_Cache -->|Read Rates| SL
        Pricing_Cache -->|Read Rates| Daemon
    end
```

### Component Breakdown

1. **CLI Renderer (`statusline`)**:
   - A ultra-fast CLI binary (executes in $<2\text{ms}$) invoked by Antigravity on every turn/state change.
   - **Deduplication:** Appends current turn tokens to a local session file (`usage_<conversation_id>_<YYYY-MM-DD>.jsonl`) if and only if the current total input/output tokens differ from the last line (preventing duplicate logs for same-state CLI triggers).
   - **Cached Token Calculation:** Since the CLI does not provide session-wide cached tokens, `statusline` scans the current session's `.jsonl` file and sums up all cached input tokens.
   - **Render Status Line:** Reads `pricing_cache.json` and `api_usage.json` to calculate turn costs, session costs, and extract today's accumulated cost and daemon health status.

2. **Background Daemon (`agy-statusline-daemon`)**:
   - A background service (running via systemd on Linux or launchd on macOS) scheduled to run every 1 minute.
   - **Pruning:** Automatically scans the cache folder and deletes any session logs older than 7 days to conserve disk space.
   - **Pricing Sync:** Downloads the latest rates from remote repository (falling back to an embedded pricing copy) to update `pricing_cache.json` once every 24 hours.
   - **Daily Aggregation:** Finds all session logs for today, aggregates token usages per model, calculates pricing rates, and atomically dumps the results to `api_usage.json`.

## JSON Schemas & Caches

All cache and state files are located inside: `~/.gemini/antigravity-cli/cache/`

### 1. `api_usage.json` (Today's Accumulated Cost)
This file is generated by the background daemon and read by the statusline renderer to show today's accumulated USD billing.

```json
{
  "last_poll_time": "2026-07-10T14:53:00+02:00",
  "status": "success",
  "error_message": "",
  "today_cost_usd": 0.1245,
  "models": {
    "gemini3.5flash": {
      "input_tokens": 1200000,
      "output_tokens": 250000,
      "cached_tokens": 300000,
      "cost_usd": 0.1245
    }
  }
}
```

#### Field Specifications:
* **`last_poll_time`** (string): The RFC3339 timestamp of the daemon's last aggregation run. Used by `statusline` to detect if the daemon has died (displays `[Daemon Dead]` if older than 5 minutes).
* **`status`** (string): The health/status of the last background run (`"success"`, `"error"`). If not `"success"`, the statusline displays `[Daemon Err]` or `[Offline]`.
* **`error_message`** (string, optional): Detailed aggregation/pricing error if daemon run failed.
* **`today_cost_usd`** (float64): The total USD cost accumulated across all active models today.
* **`models`** (map): Per-model detailed breakdown of input, output, and cached tokens consumed today, along with model-specific billing costs.

### 2. `pricing_cache.json` (Pricing Rates)
Synchronized from the remote repository once every 24 hours.

```json
{
  "last_fetched": "2026-07-10T14:53:00+02:00",
  "models": {
    "gemini-3.5-flash": {
      "input_price_per_1m": 1.5,
      "cached_input_price_per_1m": 0.15,
      "output_price_per_1m": 9.0
    }
  }
}
```

### 3. Session Log File (`usage_<conversation_id>_<YYYY-MM-DD>.jsonl`)
Appended line-by-line on every turn by `statusline`.

```json
{"timestamp":"2026-07-10T15:00:00Z","model_id":"gemini-3.5-flash","input_tokens":1200,"cached_input_tokens":300,"output_tokens":250,"total_input_tokens":45000,"total_output_tokens":3100}
```
